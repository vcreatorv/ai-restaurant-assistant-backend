# Гайд по описаниям блюд в меню

Описания блюд в `seed/menu.json` → поле `description`. Цель — помочь гостю понять, **что именно** он покупает, а не уговорить его что-то взять.

## Принципы

1. **Лаконично.** 1–2 предложения, ~120–200 знаков. Гость пробегает глазами по меню — длинный текст утомляет.
2. **Ингредиенты как точки опоры.** Особенно — экзотические, региональные, премиальные. «Кедровые орехи», «мраморная говядина Блэк Ангус», «Кастилия Ла Манча».
3. **Способ приготовления.** Если он осмысленный — называем: су-вид, томлёный, на углях хоспера, на пару, конфи, маринованный, обжаренный, запечённый. Это меняет представление гостя о блюде.
4. **Текстура и вкус.** Хрустящий, нежный, тающий, плотный, бархатный, кремовый, наваристый, насыщенный, ароматный, пряный, кисло-сладкий, дымный.
5. **Происхождение.** Где уместно — указываем: «японский даси», «итальянская классика», «Кастилия Ла Манча», «техасская подача».
6. **Без рекламы.** Слова «прекрасный», «изумительный», «восхитительный», «лучший в городе» **не использовать** — они звучат фальшиво и обесценивают описание.
7. **Без КБЖУ и пользы.** Не пишем «полезный», «диетический», «низкокалорийный». Калорийность и нутриенты — отдельные поля API. В описании говорим только о гастрономии.
8. **Без аллергенов и диет.** «Веган», «без глютена», «не содержит молочки» — отдельные структурированные поля. В тексте описания — нет.
9. **Без цены.** Цена — отдельное поле. В описании цены и сравнений по цене не упоминаем.

## «Вкусные слова» — ориентир

Опираемся на 9 измерений:
- **цвет** — золотистый, рубиновый, янтарный, изумрудный
- **размер** — большой, миниатюрный, на кости, на компанию
- **способ** — су-вид, на углях, томлёный, конфи, маринованный, ферментированный
- **сорт / порода** — Блэк Ангус, Рислинг, Арборио, темпранильо
- **бренд** — Jack Daniel’s, авторский, фирменный
- **страна / регион** — испанский, тайский, немецкий, провансальский, бразильский
- **происхождение** — фермерский, авторский, классический
- **вкус** — пряный, кисло-сладкий, насыщенный, бархатный, копчёный, сладковатый
- **текстура** — хрустящий, тающий, плотный, воздушный, кремовый

## Шаблоны по категориям

| Категория | Что подсветить |
|---|---|
| Закуски | повод (к вину/пиву / для компании / для лёгкого старта), текстура |
| Салаты | свежесть, заправка, сытность |
| Супы | наваристость, способ варки, температура подачи |
| Пицца / паста | тесто (тонкое/римское), соус, ключевые ингредиенты |
| Бургеры | отруб котлеты, соус, дополнения, булочка |
| Стейки | отруб, откорм (зерно/трава), порода, мраморность |
| Гриль (хоспер) | факт хоспера, маринад, дымный аромат |
| Горячее мясо | способ томления (су-вид/конфи), соус |
| Горячее рыба | свежесть, способ (на пару, целиком, в масле) |
| Гарниры | текстура, ароматизация, к чему уместно |
| Соусы | вкусовой профиль, основа, к чему |
| Десерты | текстура, температура подачи, акцент |
| Кофе | способ заваривания, баланс молока/кофе, нота вкуса |
| Чай | страна / тип ферментации / вкусовая нота |
| Лимонады | основные ноты, кислотность, освежающий эффект |
| Алкоголь | страна, регион, сорт винограда / тип пива, нота |

## Антипримеры

| Плохо | Почему |
|---|---|
| «Очень вкусное и сытное блюдо.» | Пустые слова, ничего не говорят о вкусе. |
| «Полезный диетический борщ из натуральных продуктов.» | КБЖУ + рекламные клише. |
| «Лучший стейк в городе!» | Эмоциональная реклама. |
| «Идеально для тех, кто следит за фигурой.» | Фитнес-нарратив, не место. |
| «Огромная порция за лучшую цену.» | Цена и оценочное суждение. |

## Хорошие примеры

> **Тартар из говядины** — Изысканная закуска из отборной вырезки с каперсами, вялеными томатами и песто, подаётся с пармезаном и хрустящим бон багетом.

> **Свиные ребра «Барбекю»** — Свиные рёбра су-вид в густом соусе барбекю с луком фри — мягкие до отделения от кости, подаются с блю чизом.

> **Ризотто с белыми грибами** — Кремовое ризотто на рисе арборио с белыми грибами, белым вином и пармезаном, обогащённое сливочным маслом по-итальянски.

## Как обновлять

1. Изменить запись в словаре `DESCRIPTIONS` в `scripts/apply_descriptions.py`.
2. Прогнать `python scripts/apply_descriptions.py` — обновит `seed/menu.json`.
3. Прогнать `go run ./cmd/seed -mode=update-descriptions` — обновит БД без переинсёрта.

Изменение текста описания влияет на embedding-вектор для RAG, поэтому после массовой правки описаний потребуется переиндексация Qdrant (см. `cmd/embed-menu`, появится в шаге A2).
